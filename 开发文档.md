# Rust + Slint 现代文件管理器开发文档

> 项目代号：**FerroxPlorer**
> 技术栈：Rust（后端逻辑）+ Slint（UI 框架）
> 目标平台：Windows
> 界面语言：简体中文（默认）

---

## 一、项目概述

### 1.1 项目目标

本项目旨在使用 Rust 语言与 Slint GUI 框架，开发一款兼具 Windows 资源管理器全部核心功能，并融合 Files（files-community/Files）项目现代设计理念的桌面文件管理器。应用以**中文为默认语言**，面向中文用户优化交互体验，同时保持对开发者友好的底层扩展性。

### 1.2 核心设计原则

- **功能完整性**：覆盖 Windows 资源管理器全部原生功能
- **现代化体验**：参考 Files App 的流畅 UI 与高密度功能（标签页、双面板、命令面板等）
- **本地性能**：Rust 原生实现，无 Electron 负担，启动迅速、内存高效
- **跨平台一致**：主攻 Windows，兼顾 Linux / macOS
- **可扩展架构**：插件系统与主题系统预留接口

### 1.3 与 Windows 资源管理器功能对照

| 功能模块 | Windows 资源管理器 | FerroxPlorer 支持 |
|---------|-----------------|-----------------|
| 文件浏览与导航 | ✅ | ✅ 增强版 |
| 地址栏 / 面包屑 | ✅ | ✅ Omnibar 统一栏 |
| 快速访问 / 收藏 | ✅ | ✅ 增强（标签固定） |

| 复制 / 粘贴 / 移动 | ✅ | ✅ 后台队列进度显示 |
| 文件属性 | ✅ | ✅ 增强（哈希 / 数字签名） |
| 回收站 | ✅ | ✅ |
| 右键上下文菜单 | ✅ | ✅ 可扩展 |
| 网络 / 云存储 | ✅ 有限 | ✅ 多云驱动支持 |

## 二、技术架构

### 2.1 整体架构分层

```
┌─────────────────────────────────────────────────┐
│               Slint UI 层（.slint DSL）           │
│   界面声明、组件绑定、主题变量、布局逻辑             │
├─────────────────────────────────────────────────┤
│               Rust 业务逻辑层                     │
│   文件系统操作、搜索引擎、标签系统、Git、云存储       │
├─────────────────────────────────────────────────┤
│               系统 API 层                        │
│   Windows API（shell32 / ntfs）、POSIX、网络 FS   │
└─────────────────────────────────────────────────┘
```

### 2.2 核心依赖库

| 类别 | 库名 | 用途 |
|------|------|------|
| UI 框架 | `slint` | 声明式 GUI 渲染 |
| 异步运行时 | `tokio` | 文件操作异步化 |
| 文件系统 | `std::fs` + `walkdir` | 目录遍历 |
| 文件监听 | `notify` | 实时目录变化监听 |
| 搜索索引 | `tantivy` | 全文/文件名索引搜索 |
| 归档操作 | `sevenz-rust` / `zip` | 压缩包读写 |
| 哈希校验 | `sha2` / `md-5` | 文件哈希计算 |
| Git 集成 | `git2` | 仓库状态读取 |
| 云存储 | `opendal` | 统一多云存储抽象 |
| 图标提取 | `winapi` / `ico` | 系统图标读取 |
| 国际化 | `slint` 内置 `@tr()` 宏 | 中文本地化 |
| 配置持久化 | `serde` + `toml` | 用户设置序列化 |
| 缩略图 | `image` + `ffmpeg-next` | 图片/视频预览 |

### 2.3 Slint 版本要求

使用 Slint **1.13+**，以获得：
- `FocusScope` 全局快捷键捕获
- `MenuBar` / `MenuItem` 原生菜单支持
- `DragArea` / `DropArea` 实验性拖放（需设置 `SLINT_ENABLE_EXPERIMENTAL_FEATURES=1`）
- 模态窗口 API（`show_modal`）
- 系统托盘支持（`SystemTray`）

## 三、功能模块详细说明

### 3.1 文件导航核心

**功能要求：**
- 支持本地磁盘、网络共享路径（UNC）、虚拟路径（此电脑、回收站、快速访问）
- 双击文件夹进入，右键→在新标签/新面板打开
- 前进 / 后退 / 上级目录导航，带历史记录下拉
- 面包屑导航可点击跳转任意层级，末段可直接编辑路径
- 支持键盘导航（方向键、Enter、Backspace）

**数据模型：**
- `NavigationHistory`：栈结构，存储前进/后退路径
- `DirectoryEntry`：单个文件/文件夹条目（名称、大小、修改时间、图标、类型、标签）
- `VirtualPath`：抽象虚拟节点（此电脑、网络位置等）

**性能要求：**
- 超大目录（10万+ 文件）需分批加载（虚拟列表渲染）
- 使用 `notify` 监听当前目录变化，实时刷新，无需手动 F5

---

### 3.2 Omnibar（统一地址/搜索/命令栏）

参考 Files v4.0 设计，将地址栏、搜索框、命令面板入口三合一。

**三种模式切换：**
1. **路径模式**（默认）：显示面包屑，可编辑绝对路径，支持路径补全
2. **搜索模式**（键入 `/` 或 `Ctrl+F`）：切换为搜索输入，实时过滤当前目录
3. **命令模式**（键入 `>`）：进入命令面板，列出可用操作与快捷键提示

**技术要点：**
- Slint `TextInput` 组件，绑定 Rust 侧路径补全逻辑
- 路径补全候选列表用 `Popup` 窗口实现
- 模式切换动画：平滑左右过渡

---

### 3.3 侧边栏导航树

**节点分类（与 Windows 资源管理器一致）：**
- **快速访问**：固定文件夹 + 最近使用（可拖拽排序）
- **主目录**：桌面、文档、下载、图片、音乐、视频
- **此电脑**：所有本地磁盘驱动器（含剩余空间进度条）
- **云存储**：OneDrive、iCloud、自定义 OpenDAL 驱动
- **网络位置**：SMB/WebDAV 共享
- **标签**：所有自定义标签节点
- **回收站**：文件数量角标显示

**交互细节：**
- 折叠 / 展开各分区
- 磁盘节点显示容量进度条（已用/总量）
- 右键节点弹出上下文菜单（固定、解除固定、在新标签打开、属性）
- 拖拽文件夹到快速访问区域可直接固定

---

### 3.4 文件内容区（多视图）

提供五种视图，用户可在工具栏切换：

| 视图 | 说明 | 适用场景 |
|------|------|---------|
| 大图标 | 256px 缩略图网格 | 图片/视频库 |
| 中等图标 | 64px 图标网格 | 通用浏览 |
| 小图标 | 32px 列表行 | 密集文件列表 |
| 详细信息 | 多列表格（名称/大小/类型/日期） | 文件管理 |
| 列视图 | Miller Columns 多列级联 | 快速深层导航 |

**通用交互：**
- 单击选中，Shift/Ctrl 多选，框选（鼠标拖拽矩形选择）
- 双击打开文件（调用系统默认程序）或进入文件夹
- 右键弹出上下文菜单
- F2 重命名（行内编辑），Enter 确认，Escape 取消
- Delete 移到回收站，Shift+Delete 永久删除（需确认对话框）
- 拖放：同盘移动，跨盘复制，Shift 拖拽强制移动，Alt 拖拽强制复制

**虚拟列表渲染：**
- Slint `ListView` / `GridView` 绑定 Rust `VecModel`
- 超过 500 条目启用虚拟化，仅渲染可视区域
- 懒加载缩略图（后台线程计算，完成后更新模型）

---

### 3.5 标签页系统

**功能：**
- `Ctrl+T`：新建标签页（默认打开"快速访问"）
- `Ctrl+W`：关闭当前标签页
- `Ctrl+Tab`：切换标签页
- 双击标签栏空白处：新建标签页
- 标签页可拖拽重排
- 右键标签页菜单：关闭、关闭其他、复制标签、固定标签
- 固定标签页宽度收缩，仅显示网站图标

---

### 3.6 双面板模式

**激活方式：** 工具栏按钮或 `F3`

**布局：**
- 左右两个独立文件面板，各有独立的标签页、地址栏、导航历史
- 可调节分隔线（拖拽改变宽度比例）
- 工具栏增加 "镜像路径"（右面板跟随左面板路径）、"互换面板" 操作
- 跨面板拖拽文件，默认为移动操作

---

### 3.7 搜索与过滤

**搜索行为：**
- **实时过滤**（当前目录内容过滤，不递归）：Omnibar 搜索模式下键入即过滤
- **全局索引搜索**（递归深层搜索，使用 Tantivy 索引）：触发完整搜索结果面板

**搜索结果面板功能：**
- 支持高级语法：`name:*.rs`、`size:>1MB`、`modified:today`、`tag:重要`
- 按名称、路径、大小、日期排序
- 结果分组（文件夹 / 文档 / 图片 / 其他）
- 点击结果：定位至父文件夹并选中文件

---

### 3.8 文件操作与进度

**操作类型：**
- 复制（Ctrl+C / 拖拽跨盘）
- 移动（Ctrl+X+V / 拖拽同盘）
- 删除（Delete / Shift+Delete）
- 重命名（F2）
- 创建（Ctrl+Shift+N 新建文件夹，右键→新建→文件类型）
- 解压缩 / 压缩（右键菜单集成）

**操作队列与进度：**
- 所有耗时操作（复制/移动大文件）进入后台 Tokio 任务
- 状态栏显示实时进度（当前文件、速度、剩余时间）
- 支持暂停、取消单个操作
- 操作完成后状态栏显示摘要（"已复制 N 个文件"）
- 同名文件冲突弹出解决对话框（覆盖/跳过/重命名/全部应用）

---

### 3.9 文件属性面板

**内联详情面板（右侧可折叠）：**
- 实时预览选中文件（图片缩略图、文本前 N 行、媒体元数据）
- 显示文件路径、大小、创建/修改日期、只读/隐藏属性

**属性对话框（双击属性 / Alt+Enter）：**
- **常规**：文件名、类型、位置、大小、日期、属性复选框
- **安全**：文件权限（NTFS ACL，Windows）
- **详细信息**：扩展元数据（EXIF、MP3 标签、Office 文档信息）
- **哈希校验**（参考 Files v4.0）：MD5 / SHA-1 / SHA-256 / SHA-512 计算与复制
- **数字签名**（Windows）：显示可执行文件的代码签名信息

---

### 3.10 标签（Tag）系统

**功能：**
- 用户可为任意文件/文件夹添加颜色标签和文字标签
- 标签存储于独立 SQLite 数据库（不修改文件系统）
- 文件图标上叠加彩色点标记
- 侧边栏"标签"节点下按标签分组浏览
- 搜索支持 `tag:` 前缀语法

---

### 3.11 Git 集成

**功能（参考 Files v4.0 Developer Tools）：**
- 自动检测目录是否为 Git 仓库（向上查找 `.git`）
- 文件状态图标叠加（已修改/未追踪/已暂存/冲突）
- 侧边栏显示当前分支名
- 右键菜单增加：在终端打开、在 IDE 打开（可配置）
- 支持 URL 拖拽克隆（拖入 GitHub URL 自动触发 `git clone`）

---

### 3.12 归档管理

**功能：**
- 支持格式：ZIP / 7z / TAR / GZ / BZ2 / XZ / RAR（读取）
- 直接浏览归档内容（无需解压，虚拟文件夹视图）
- 右键→解压到当前文件夹 / 解压到子文件夹 / 选择目标路径
- 选中文件→右键→压缩为→选择格式与压缩级别
- 归档操作支持后台进度显示

---

### 3.13 云存储集成

使用 `opendal` 统一抽象层，支持以下服务：
- OneDrive（MSAL 认证）
- 阿里云 OSS
- 腾讯 COS
- Amazon S3 / 兼容服务
- WebDAV（坚果云等）
- SFTP

**显示方式：**
- 侧边栏"云存储"分区独立节点
- 与本地文件夹一致的浏览体验
- 下载缓存机制（首次访问缓存至本地临时目录）
- OneDrive 属性面板显示同步状态与配额使用量

---

### 3.14 命令面板

`Ctrl+K` 或 Omnibar 键入 `>` 触发。

**功能：**

- 可从命令面板直接触发文件操作

## 四、界面设计规范

### 4.1 视觉风格

- **设计语言**：现代 Fluent Design，参考 Files App 与 Windows 11 设计规范
- **布局密度**：中等密度（行高 36px，图标间距 8px）
- **圆角**：组件圆角 6px，对话框 10px，按钮 4px
- **亮色/暗色模式**：系统自动跟随，可手动切换
- **主题色**：用户可自定义（默认蓝色系）
- **字体**：微软雅黑（Windows）/ PingFang SC（macOS）/ Noto Sans CJK SC（Linux）

### 4.2 颜色变量（Slint 主题）

在 `theme/colors.slint` 中定义，分亮/暗两套：

- `--背景色`：主窗口背景
- `--表面色`：侧边栏、面板背景
- `--边框色`：分隔线、输入框边框
- `--主色调`：选中高亮、强调按钮
- `--文字-主要`：正文
- `--文字-次要`：元数据、时间戳
- `--危险色`：删除、错误状态
- `--成功色`：操作完成、已同步状态

### 4.3 图标规范

- 所有内置 UI 图标使用 SVG 格式，存放于 `assets/icons/`
- 文件类型图标从系统提取（Windows: `SHGetFileInfo`），回退到内置图标集
- 图标尺寸规格：16px / 24px / 32px / 64px / 128px / 256px
- 工具栏图标：24px，侧边栏节点：16px，文件网格大图标：128px/256px

以下为各核心界面的 AI 设计提示词，可用于 UI 设计工具（Figma AI、Midjourney 等）或代码生成助手。

---

### 5.1 主窗口 (`main_window.slint`)

```
设计一个现代桌面文件管理器的主窗口界面，中文界面，Windows 11 Fluent Design 风格。

布局从上到下分为：
```

## 六、国际化（i18n）实现规范

### 6.1 中文字符串管理

所有 UI 文本使用 Slint 内置 `@tr("字符串")` 宏包裹，支持运行时切换语言。

字符串文件格式采用 Fluent（`.ftl`），存放于 `assets/i18n/zh-CN.ftl`。

命名规范：
- 界面区域前缀：`sidebar-`, `toolbar-`, `menu-`, `dialog-`, `settings-`
- 操作动词：`action-copy`, `action-paste`, `action-delete`
- 状态描述：`status-copying`, `status-ready`, `status-no-items`

### 6.2 默认语言

应用启动时读取系统语言，简体中文（`zh-CN`）下直接使用中文字符串；其他语言回退到英文。用户可在设置中手动覆盖语言。

## 七、性能规范

### 7.1 启动性能目标

- 冷启动至首屏可交互：< 800ms
- 热启动：< 300ms
- 首次加载目录（1000 文件）：< 200ms

### 7.2 内存目标

- 空载内存占用：< 60MB
- 浏览 10,000 文件目录：< 150MB
- 缩略图缓存：LRU 缓存，最大 128MB，可配置

### 7.3 关键路径优化

- 目录列表加载：`tokio::spawn` 异步读取，UI 线程不阻塞
- 缩略图生成：后台线程池（`rayon`），按需生成，结果缓存
- 搜索索引：应用启动后台建立，增量更新（基于 `notify` 事件）
- Slint 虚拟列表：仅实例化可视区域条目，条目数量不影响渲染性能

## 八、安全与权限

- 文件操作异常统一捕获，显示友好中文错误提示（无裸露的系统错误码）
- 永久删除操作必须弹出确认对话框，明确标红"此操作不可撤销"
- 对系统目录（C:\Windows 等）操作时显示额外警告
- 云存储凭据使用系统密钥链（Windows Credential Manager / macOS Keychain）存储，不明文写入配置文件
- 在 Windows 上以标准用户权限运行，需要管理员权限时通过 UAC 提权提示

### 10.1 构建

```bash
# Windows 发布构建
cargo build --release --target x86_64-pc-windows-msvc

# 打包（使用 cargo-bundle 或 WiX）
cargo bundle --release
```

### 10.2 分发格式

### 10.3 版本规范

遵循语义化版本（SemVer）：`主版本.次版本.修订号`
- 主版本：破坏性 API 或架构变更
- 次版本：新增功能
- 修订号：Bug 修复与性能优化
